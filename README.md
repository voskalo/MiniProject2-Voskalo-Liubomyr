# MiniProject2-Voskalo-Liubomyr
This is a GitRepo for Miniproject 2 of OOP in UCU from Voskalo Liubomyr. All the code from the course is here.

**Курс:** [Claude Code in Action: Master AI-Powered Development](https://www.google.com/search?q=https://claudecertifications.com/courses/claude-code-in-action/)

Цей репозиторій є технічним звітом та збіркою практичних матеріалів, напрацьованих під час вивчення курсу по роботі з Claude Code CLI.

**Організація матеріалів:**

* **Формат файлів:** Усі конспекти мають розширення **.sh**. Це зумовлено тим, що процес навчання та взаємодії з асистентом відбувався виключно в інтерфейсі командного рядка (CLI) на MacBook Air. Кожен файл є логом термінальної сесії.
* **Діалоги з AI:** Повні версії розмов розміщені в папці `conversations`.
* **Інструкція для ознайомлення:** Перед переглядом діалогів та кодів необхідно ознайомитися з файлом **read_at_first.txt**, де описана логіка взаємодії та технічні передумови сесій.

## Детальний розбір модулів та засвоєного матеріалу

### Модуль 1: Архітектура та базові інструменти

* **Урок 1: Концепція Coding Assistant.** Зрозумів різницю між стандартними LLM та агентними воркфлоу. Ключова цінність: Claude Code має прямий доступ до файлової системи, що виключає необхідність ручного копіювання коду в чат.
* **Урок 2: Інструментарій у дії.** Вивчено використання вбудованих інструментів. Це дозволяє асистенту самостійно проводити аудит репозиторію та знаходити помилки в логіці.

### Модуль 2: Контекстне управління

* **Урок 3: Налаштування середовища.** Пройдено етап встановлення CLI через `curl` та авторизацію через OAuth. Налаштовано робочий простір на MacOS.
* **Урок 4: Ініціалізація та CLAUDE.md.** Навченося створювати базу знань проєкту. Файл `CLAUDE.md` виступає «точкою істини» для ШІ, визначаючи правила кодування.
* **Урок 5: Планування та Thinking Mode.** Опановано режими `Planning` для проектування змін та `Ultrathink` для складного аналізу. Це критично для обґрунтування важкиз задач та створення плану
* **Урок 6: Контроль історії.** Вивчено команди `/compact` для стиснення історії та `/clear` для повного очищення контексту, що запобігає «галюцинаціям» моделі при зміні задачі.

### Модуль 3: Розширення та автоматизація

* **Урок 7: Custom Commands.** Реалізовано власні команди автоматизації. Побачив, як створити скрипт `/audit`, який об'єднує запуск лінтерів, тестів та перевірку безпеки пакетів.
* **Урок 8: MCP Servers.** Вивчено протокол Model Context Protocol. Навчився підключати сервер, що дає Claude можливість виконувати прямі запити до бази даних проєкту.
* **Урок 9: GitHub Integration.** Автоматизовано життєвий цикл розробки: від створення гілок у Git до формування Pull Requests та моніторингу статусів GitHub Actions.
* **Урок 10: Теорія Hooks.** Засвоєно принципи життєвого циклу команд (PreToolUse та PostToolUse). Це дозволяє запускати власні перевірки до або після дій ШІ.
* **Урок 11-12: Впровадження хуків.** Навчився реалізовувати автоматичне форматування. Тепер при будь-якій зміні файлу Claude автоматично може запускати `black` та `isort`, забезпечуючи дотримання PEP8.
* **Урок 13: Шаблони безпеки.** Зрозумів, як налаштовувати хуки, що блокують деструктивні bash-команди, наприклад, видалення бази даних чи критичних конфігурацій.
* **Урок 14: Claude Code SDK.** Побачив, як виглядає програмний інтерфейс для інтеграції агента в Python-скрипти, що дозволяє будувати автономні системи обслуговування коду.
